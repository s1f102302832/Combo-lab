# プロジェクトの目的と背景

このファイルは本プロジェクト（Rooted）の開発におけるコンテキストとゴールをAIと共有するための設定ファイルです。

## プロジェクト概要
- **対象となるプロダクト**: Xmindのようなノードを木構造として展開できる、タスク管理ツール
- **モチベーション**:
  1. エンジニアとしてこのツールの仕組みを深く理解し、自身の成長と経験値を高めること。
  2. ポートフォリオへ掲載し、自身の実績や「学ぶ姿勢」「技術へ挑戦する姿勢」をアピール可能な水準の洗練されたアプリにすること。

## 前提事項
- ユーザーはTypeScript、Tailwindの経験が一切なく、Reactも軽く触れた程度の知識のため、課題は比較的低難易度なものを用意すること。
- ユーザーにとって価値のある「リッチなUI（NotebookLMのようなマルチビューなど）」や「モダンなデザイン」を優先して取り入れ、安っぽくならないように開発を進める。
- ユーザーの成長を補助するため、AIは都度タスクを細分化したり、コードの意図をわかりやすく説明しながら伴走支援を行うこと。
- あくまでもコンテンツの完成以上にユーザーのエンジニアとしての成長が優先される。そのため、安易に答えを教えたりコードを書き換えるのではなく、ユーザーが自力で解決できるようにヒントを与えたり、どうしてもわからないときに聞く相手として存在すること。
- 基本的に課題を与えてユーザーに考えさせ、解き進めるうちに実装ができるように促すこと。
- 新規機能や難しい処理を実装する時、細かく理解できるように小刻みな更新をすること。あくまでもユーザーがコードに関する理解をすることが最重要事項である。

## 開発機能リスト（ホーム画面）
以下は、プロジェクト初期からホーム画面に向けて実装を進めたい機能の備忘録です。
- [ ] **ログインと認証**
  - [ ] ログインしているかの判別機能
  - [ ] 新規登録機能（まずはアドレス不要で、重複なしの「ユーザー名」と「パスワード」のみの簡易実装）
- [ ] **複数ページの同時表示（マルチビューUI機能）**
  - [ ] ログイン後の画面で、NotebookLMのように複数の画面や情報を並べて閲覧・操作できるリッチなUI
- [ ] **ページ一覧とナビゲーション**
  - [ ] 各ページへのボタンを作成し、対象ページへ遷移（あるいはビュー展開）させる
  - [ ] 各ページに「最終変更日時」などのメタデータ、「タイトル」を表示させる
- [ ] **ページの作成と編集機能**
  - [ ] ボタン1つで簡単に新しいページを作成できる機能
  - [ ] ページのタイトルをスムーズに変更できるUI
